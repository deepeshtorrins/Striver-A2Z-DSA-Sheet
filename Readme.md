# DSA Learning Plan (Striver A2Z)

## 1. Read the Problem

* Spend **5-10 minutes** thinking about the problem.
* Ask yourself:

  * What is the brute force approach?
  * Can I optimize it?
* Try to come up with a solution before watching the video.

---

## 2. Watch Striver's Explanation

### While Watching

❌ Don't copy every line of code.

✅ Write only the important points:

```text
Problem Name

Brute Force
TC:
SC:

Better
TC:
SC:

Optimal
TC:
SC:

Observation:
```

---

## 3. Close the Video

* Write the code yourself in VS Code.
* Do not copy-paste.
* Even if you understood the solution, implement it from memory.
* If you cannot write it without looking, revisit the explanation and try again.

---

## 4. Next Day Revision

This is the most important step.

Before learning a new topic:

* Spend 20 minutes revising yesterday's problems.
* Re-code them without looking at the solution.

### Example

Today:

* Missing Number
* Move Zeros
* Union of Arrays

Tomorrow:

* Rewrite all 3 problems from memory.

# Notes Template

Do not write detailed notes for all problems.

For each problem, write:

```text
Problem:

Pattern:

Observation:

Brute Force:
TC:
SC:

Optimal:
TC:
SC:

Mistake I Made:
```

### Example

```text
Problem:
Count Subarray Sum = K

Pattern:
Prefix Sum + HashMap

Observation:
If (prefixSum - k) exists before,
then a subarray with sum k exists.

Brute Force:
TC: O(n²)
SC: O(1)

Optimal:
TC: O(n)
SC: O(n)

Mistake I Made:
Forgot to initialize mpp[0] = 1.
```

# Weekly Schedule (Working Professional)

## Monday - Friday

* Learn 1 concept.
* Solve 1-2 problems.
* Write short notes.
* Revisit previous problems.

## Saturday

* Revise all problems solved during the week.
* Re-code without looking.

## Sunday

* Learn a new topic.
* Solve 2-3 problems.
* Review mistakes and weak areas.

---

# Goal

❌ Do not focus on completing 474 problems quickly.

✅ Focus on:

* Understanding patterns.
* Writing code from memory.
* Explaining the logic in your own words.
* Consistent progress every week.

The goal is not to finish the sheet. The goal is to become confident in solving unseen problems.
